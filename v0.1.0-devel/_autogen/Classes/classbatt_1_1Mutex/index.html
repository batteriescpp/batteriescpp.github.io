    Detailed Description🔗

    1
2
    template <typename T >
class batt::Mutex;

    Provides mutually-exclusive access to an instance of type T.

    This class has two advantages over std::mutex:

    1. It will yield the current batt::Task (if there is one) when blocking to acquire a lock, allowing the current thread to be used by other tasks
    2. By embedding the protected type T within the object, there is a much lower chance that state which should be accessed via a mutex will accidentally be accessed directly This mutex implementation is mostly fair because it uses a modified version of Lamport's Bakery Algorithm
