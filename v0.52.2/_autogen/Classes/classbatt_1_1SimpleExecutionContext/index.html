    batt::SimpleExecutionContext

    batt::SimpleExecutionContext🔗

    A minimal multi-threaded execution context implementation, suitable for parallel-compute thread pools. #include <batteries/async/simple_executor.hpp>

    Inherits from boost::asio::execution_context

    function stop🔗

    1
    inline auto stop()

    Signals to all threads inside this->[run()]() to exit their event processing loop ASAP.

    Once all threads have exited [run()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-run), this->[reset()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-reset) must be called before this->[run()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-run) to resume normal processing.

    function is_stopped🔗

    1
    inline auto is_stopped() const

    Returns true as soon as this->[stop()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-stop) is called; this does not indicate that threads have exited this->[run()](), only that this->[stop()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-stop) has been called.

    Initially false when a context is constructed.

    function run🔗

    1
    inline auto run()

    Blocks the current thread processing [post()]()-ed completion handlers while all of the following are true:

    Return: The number of handlers processed.

    • This context's work count is non-zero
    • this->[is_stopped()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-is-stopped) is false

    function reset🔗

    1
    inline auto reset()

    Sets this->[is_stopped()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-is-stopped) to false to allow threads to enter this->[run()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-run) to resume work processing.

    This should only be called once the caller is sure that no thread is inside this->[run()](/_autogen/Classes/classbatt_1_1SimpleExecutionContext/#function-run).
